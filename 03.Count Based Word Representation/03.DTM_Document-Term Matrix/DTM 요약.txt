문서 단어 행렬(Document-Term Matrix, DTM) 요약

문서 단어 행렬이란?
- 서로 다른 문서들의 BoW들을 결합한 표현 방법
- 다수의 문서에서 등장하는 각 단어들의빈도를 행렬로 표현한 것
- 각 문서에 대한 BoW를 하나의 행렬로 만든 것
- BoW의 표현을 다수의 문서에 대해서 행렬로 표현하고 부르는 용어
- 내가 이해한 바로는...
-> 모든 문서에서 단어단위로 쪼갠 BoW를 모두 모아서 
-> 각 단위가 출현한 빈도수대로 표현하는 것
-> 행에는 문서번호, 열에는 각 단어로 표현할 수 있음

문서 단어 행렬의 한계

1. 희소 표현
- 기본적으로 원-핫 벡터와 유사하기에 공간적 낭비와 계산 리소스를 증가시킬 수 있음
- 벡터의 대부분이 0을 값을 가지게 됨
-> 예를 들어서, 의학 논문과 요리책 데이터를 가지고 문서 단어 행렬을 표현한다면...
-> 의학 논문, 요래책에서 나온 단어들 모두 많은 '0'데이터를 가지고 있을 것임
-> 왜냐? 데이터가 안겹치니까!!
- 해결 방법으로는 
-> 전처리를 통해 단어 집합의 크기를 줄이기

2. 단순 빈도 수 기반 접근
- 단순 빈도 수 기반 접근이기에 각 문서의 유사성을 확인 해 볼 수 없음
- 예) like나 the라는 단어의 빈도수가 많다고 하여 각 문서의 유사성 확인 x
- 해결방법
-> DTM에 불용어와 중요한 단어에 대해 가중치를 주는 방법
-> TF-IDF 방법! 다음 단원!
