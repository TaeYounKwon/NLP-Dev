케라스 API 및 서브클라싱 요약

케라스에는...
- Sequential API
- 장점:
-> 간단한 모델을 구현하기에 적합함
- 단점: 
-> 다수의 입력, 다수의 출력을 가진 모델 
-> 층간의 연결이나 덧셈과 같은 연산을 하는 모델을 구현하기에는 쉽지 않음

- Functional API
- 장점:
-> Sequential API로 구현할 수 없는 복잡한 모델들을 구현 가능
- 단점:
-> 입력의 크기를 명시한 입력층이 모델의 앞단에 정의해줘야함
- 특징:
-> 기본적으로 딥 러닝 모델을 DAG(Directed Acyclic Graph)로 취급
-> 실제 대부분의 딥 러닝 모델이 DAG에 속함
-> 재귀 네트워크와 RNN은 DAG에 속하지 않아 Functional API로 구현이 힘듬

- Subclassing API
- 장점:
-> Functional API로 구현할 수 없는 더 복잡한 모델들을 구현 가능
- 단점:
-> 객체 지향 프로그래밍에 익수해야 하므로 코드 사용이 가장 까다로움
- 특징:
-> 밑바닥부터 새로운 수준의 아키텍처를 구현해야 하는 실험적 연구를 하는 연구자들에게 적합
