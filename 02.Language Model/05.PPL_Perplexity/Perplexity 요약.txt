Perplexity 요약

Perplexity란
- 언어모델을 평가하기 위한 평가 지표. 
- perplexed - '헷갈리는'라는 뜻, perplexity는 헷갈리는 정도로 이해
- 컴퓨터가 헷가리는 정도를 수치화시킨 것이니, 수치가 낮을수록 모델이 좋은 것!

분기 계수(Branching factor)
- PPL은 선택할 수 있는 가능한 경우의 수를 의미하는 분기계수임
- PPL은 이 언어 모델이 특정 시점에서 평균적으로 몇 개의 선택지를 가지고 고민하는지를 의미
- 두 언어 모델을 비교하는 중, 한개의 언어모델이 10, 다른 한개의 언어모델이 4의 PPL을 가지고 있다면...
-> 더 적게 나온 2번째 모델의 성능이 더 좋다고 볼 수 있음
- 단, PPL이 낮다는 것 = 테스트 데이터 상에서 높은 정확도 != 사람이 느끼기에 좋은 언어모델
- 두개의 언어모델을 비교할 때에는...
-> 두개의 언어모델 도메인에 잘 맞는 테스트 데이터를 잘 골라야 신뢰도 및 정확도 증가
